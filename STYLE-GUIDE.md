# Location Modifier Style Guide

*Standardized formatting rules for referencing modifiers across all Path campaign documents*

## **Core Formatting Rules**

### **1. Modifier Names**
- Always capitalize modifier names: **Stable**, **Underground**, **Cave Mouth**, **Wild Beasts**, **Inhabited**
- Use **bold** formatting when first introducing a modifier or when emphasizing its mechanical importance
- Use consistent terminology (never vary between "Underground" and "underground network")

### **2. Stat Block Format**
```
> **Modifiers:** [Primary], [Secondary], [Tertiary]
```

**Examples:**
```
> **Modifiers:** Stable
> **Modifiers:** Unstable, Cave Mouth
> **Modifiers:** Unstable, Underground, Wild Beasts
```

### **3. Feature Description References**

#### **Static Modifier References:**
```
"This location has the [Modifier Name] modifier"
"Due to the [Modifier Name] modifier, [effect]"
```

**Examples:**
- "This location has the **Cave Mouth** modifier"
- "Due to the **Underground** modifier, this location connects only to other Underground locations"

#### **Dynamic Modifier Changes:**
```
"gains the [Modifier Name] modifier"
"loses the [Modifier Name] modifier"
"becomes [Modifier Name]"
```

**Examples:**
- "Highland Watch gains the **Stable** modifier"
- "If the counter drops below 10, the location loses the **Stable** modifier"
- "The location becomes **Unstable**"

### **4. Cross-Reference Format**

When referencing modifiers in relation to other locations:
```
"[Action] connects to [Modifier Name] locations"
"Only accessible from [Modifier Name] locations"
```

**Examples:**
- "Provides access to **Underground** locations"
- "Only accessible from **Cave Mouth** locations"
- "Can connect to any **Stable** location the party has visited"

## **Modifier-Specific Guidelines**

### **Stable/Unstable**
- Always presented as mutually exclusive states
- Use "becomes **Stable**" not "becomes a **Stable** location"
- Conditional stability: "gains the **Stable** modifier when [condition]"

### **Underground**
- Always capitalized, never "underground network" or "underground system"
- Standard phrase: "Due to the **Underground** modifier, this location connects only to other **Underground** locations or surface **Cave Mouth** locations"

### **Cave Mouth**
- Two words, always capitalized
- Standard phrase: "This location has the **Cave Mouth** modifier - [specific entrance description]"

### **Wild Beasts**
- Two words, both capitalized
- Standard phrase: "Due to the **Wild Beasts** modifier, this location always contains a dangerous creature encounter"

### **Inhabited**
- Standard phrase: "Due to the **Inhabited** modifier, this location always contains NPCs [description of inhabitants]"

## **Common Mistakes to Avoid**

### **❌ Incorrect Formatting:**
- "cave mouth modifier" (not capitalized)
- "Underground network" (inconsistent terminology)
- "Stable location" (location vs modifier)
- "underground modifier" (not capitalized)

### **✅ Correct Formatting:**
- "**Cave Mouth** modifier"
- "**Underground** modifier"
- "**Stable** modifier"
- "**Underground** modifier"

## **Integration Examples**

### **Feature Description Integration:**
```
***Central Well - Passive:*** This location has the **Cave Mouth** modifier - the well provides access to underground tunnel systems. When PCs descend into the well, they can travel to any **Underground** location or return to any surface **Cave Mouth** location.
```

### **Conditional Modifier Changes:**
```
***Cairn Bindings - Passive:*** Each time PCs build a cairn at this highland position, mark progress on a counter (start at 0). When the counter reaches 10, Highland Watch gains the **Stable** modifier. If the counter drops below 10 for any reason, the location loses the **Stable** modifier and becomes **Unstable**.
```

### **Mechanical Integration:**
```
***Predator Territory - Passive:*** Due to the **Wild Beasts** modifier, this location always contains a dangerous predator encounter. The bone-scattered chambers serve as hunting grounds for creatures that use the fungal maze for ambush tactics.
```

## **Template Phrases by Context**

### **When Introducing a Modifier:**
- "This location has the **[Modifier]** modifier"
- "Due to the **[Modifier]** modifier, [mechanical effect]"

### **When Modifiers Change:**
- "gains the **[Modifier]** modifier"
- "loses the **[Modifier]** modifier and becomes **[New Modifier]**"

### **When Referencing Modifier Effects:**
- "Due to the **[Modifier]** modifier, [effect description]"
- "The **[Modifier]** modifier means [mechanical consequence]"

### **When Connecting Locations:**
- "connects to other **[Modifier]** locations"
- "accessible from **[Modifier]** locations"
- "travel to any **[Modifier]** location"
