# Create Chime Command

You are helping the user create crystal wind chime memories for the Starfall City in "The Path" campaign.

## What is a Chime?

Crystal wind chimes are the primary storytelling mechanic of the Starfall City. The plague crystal absorbed emotionally significant moments and condensed them into wind chimes over decades. When players touch a chime, they see a memory.

Each chime must:
- Land **one emotional note** (grief, warmth, horror, rage, love, fear)
- **Point forward** — give players a reason to look somewhere else
- Stay **sparse** — one image, let it speak

## Process

### Step 1: Establish the Scene

Ask:
- Which district?
- Which thread?
- Which characters appear?
- What is the **one emotional note** this chime should land?

### Step 2: Draft the Memory

Write the scene as bullet points — present tense, sparse. Use the character's camera direction as a constraint.

**Rules:**
- 3-5 bullet points maximum
- One image per bullet
- No exposition — show, don't explain
- End on the emotional note, not after it

**Template:**
```
### [Chime Name]
**District:** [X,X — Location Name]
**Thread:** [Which story thread]
**Characters:** [Who appears]
**Camera:** [Direction]

- [What players see]
- [What players see]
- [What players see]

**Points toward:** [Next location or discovery in the thread]
**Pointer:** [Physical object or detail that directs players — must be specific]
**Placement:** [Where exactly in the district the chime hangs — players find it by exploring]
```

### Step 3: The Pointer

Every chime needs a pointer — something players can follow to the next beat. The pointer must be **specific and legible**: players see it and know where to go next.

Good pointers:
- A key with a room number etched in
- A name on a door players will find elsewhere
- A letter with an address
- An object players will recognize when they reach the next location

Bad pointers:
- "A sense of unease pointing east"
- "Something familiar about the university"
- Vague emotional pulls

### Step 4: Iterate

Present the draft. Ask:
- Does the emotional note land?
- Is the camera direction felt in the writing?
- Is the pointer specific enough?
- Does it point clearly to the next beat?

Refine until both agree it works. Then move to the next chime.

### Step 5: Update Location Notes

After each chime is locked, update the **Location notes** block for that district in `starfall-city-stories.md`. Add any physical details the chime establishes — buildings, rooms, objects, layout — that will need to be present when the location is specced properly.

### Step 6: Thread Check (after every 3 chimes)

Pause and check the thread so far:
- Does the emotional progression make sense?
- Are the pointers forming a clear chain?
- Is the thread building toward its end point and reward?

## Scene Rulings

- Each chime does **one thing** — emotional note OR connection reveal OR reframe. Not all three.
- Threads form a **full picture** if players explore everything — no permanent gaps.
- The merchant thread uses **minimal chimes** — mostly physical evidence.
- Chimes are **not fragile** — players can return and replay them.
- Chimes are **placed objects** — players find them by exploring, the GM does not signal them.
- Each district's chimes should be noted in the **Location notes** section of that district in `starfall-city-stories.md`.

## Important Guidelines

**DO:**
- Read all four context files before starting
- Stay sparse — resist the urge to explain
- Trust the camera direction to carry emotional weight
- Make pointers specific and physical
- Work one chime at a time with user input
- Check thread coherence every few chimes

**DON'T:**
- Start without reading the context files
- Write more than 5 bullet points
- Add exposition or backstory in the scene itself
- Use vague pointers ("a feeling," "something drawing them")
- Write multiple chimes without user input
- Forget that the GM reads this memory aloud — keep it performable
