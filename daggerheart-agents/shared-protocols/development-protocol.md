# Development Protocol for The Path Campaign
## Mandatory Process for All Content Creation

---

## **CRITICAL REQUIREMENT**

**Before creating ANY content for The Path campaign, you MUST follow this step-by-step approval process:**

---

## **Step-by-Step Process**

### **1. OUTLINE FIRST**
- Present the proposed content structure
- Identify what type of content you're creating
- Explain how it fits into the campaign
- **DO NOT write the actual content yet**

### **2. IDENTIFY ALL GAME MECHANICS**
- List every mechanical element you plan to include
- Reference existing Daggerheart systems you're building on
- Explain how mechanics interact with core game rules
- Note any new dice rolls, resource costs, or conditions

### **3. REQUEST EXPLICIT APPROVAL**
Use this exact format:
```
"I'm planning to create [CONTENT TYPE] that includes [SPECIFIC MECHANICS].
Here's my outline: [STRUCTURE].
Should I proceed with this approach, or would you like me to modify the mechanics/structure first?"
```

### **4. WAIT FOR CONFIRMATION**
- **DO NOT create content until user says "proceed" or "yes"**
- If user suggests changes, revise the outline first
- Only begin writing after explicit approval

---

## **Special Focus: Game Mechanics**

### **Mechanics Requiring Extra Scrutiny:**
- New dice rolling mechanics
- Resource costs (Hope, Fear, Stress, HP)
- Condition effects or status changes
- Scaling formulas or tier progressions
- Cross-system interactions
- Features that affect spotlight or action economy

### **Validation Requirements:**
- **Reference Existing Systems**: Show which Daggerheart mechanics you're building on
- **Explain Interactions**: How new rules work with core game systems
- **Design Principle Compliance**: Verify against Daggerheart's 7 core principles
- **Balance Justification**: Why these mechanics are appropriate for their tier
- **Asymmetrical Design**: Ensure PC vs adversary features target correct mechanics

---

## **Content Categories Requiring Protocol**

### **Always Requires Step-by-Step Approval:**
- Environments with custom features
- Adversaries with new abilities
- Equipment with special properties
- Custom spells or domain cards
- Campaign-specific rules systems
- NPCs with mechanical abilities

### **May Require Approval:**
- Pure narrative content (if it implies mechanics)
- Location descriptions (if they include environmental effects)
- Background lore (if it affects character abilities)

---

## **The Path Campaign Specific Guidelines**

### **Path Generation Mechanics:**
- Must integrate with existing Path generation system
- Should follow established tier scaling (1-4)
- Cannot break the Path network's fundamental rules

### **Mist Interaction:**
- Must respect established Mist zones and effects
- Follow Mist Mark accumulation rules
- Cannot bypass Sacred Flame protection principles

### **Village Integration:**
- Consider how content affects Village in the Mist
- Respect established NPCs and faction dynamics
- Maintain campaign's themes and tone

---

## **Failure to Follow Protocol**

If you create content without following this process:
1. **STOP immediately**
2. **Present the outline** you should have shown first
3. **Wait for approval** before continuing
4. **Revise as requested**

---

## **Quick Reference Checklist**

Before creating any content, ask yourself:
- [ ] Have I outlined the structure first?
- [ ] Have I identified all game mechanics?
- [ ] Have I requested explicit approval?
- [ ] Has the user said "proceed" or "yes"?
- [ ] Am I following Daggerheart design principles?
- [ ] Does this fit The Path campaign themes?

**If ANY answer is "no" - STOP and follow the protocol.**

---

*This protocol ensures quality control, maintains game balance, and respects the collaborative nature of campaign development.*
